# Database Schema

The portfolio application uses a relational Database (MySQL).

## Tables

### `users`
- `id` (BIGINT, Primary Key, Auto Increment)
- `username` (VARCHAR, Unique, Not Null) - Admin username
- `password` (VARCHAR, Not Null) - BCrypt hashed password
- `role` (VARCHAR) - e.g., "ROLE_ADMIN"

### `projects`
- `id` (BIGINT, Primary Key)
- `title` (VARCHAR)
- `description` (TEXT)
- `image_url` (VARCHAR)
- `tech_stack` (VARCHAR) - Comma-separated or JSON
- `github_link` (VARCHAR)
- `live_link` (VARCHAR)

### `skills`
- `id` (BIGINT, Primary Key)
- `name` (VARCHAR)
- `category` (VARCHAR) - e.g., "Frontend", "Backend", "AI/ML"
- `percentage` (INT)
- `icon` (VARCHAR) - React Icon name or image URL

### `internships`
- `id` (BIGINT, Primary Key)
- `company` (VARCHAR)
- `role` (VARCHAR)
- `duration` (VARCHAR)
- `description` (TEXT)

### `certifications`
- `id` (BIGINT, Primary Key)
- `name` (VARCHAR)
- `organization` (VARCHAR)
- `date` (VARCHAR)
- `link` (VARCHAR)

### `achievements`
- `id` (BIGINT, Primary Key)
- `title` (VARCHAR)
- `type` (VARCHAR) - e.g., "Hackathon", "Award"
- `date` (VARCHAR)
- `description` (TEXT)

### `blogs`
- `id` (BIGINT, Primary Key)
- `title` (VARCHAR)
- `content` (LONGTEXT)
- `image_url` (VARCHAR)
- `tags` (VARCHAR)
- `publish_date` (DATETIME)

### `contact_messages`
- `id` (BIGINT, Primary Key)
- `name` (VARCHAR)
- `email` (VARCHAR)
- `message` (TEXT)
- `date` (DATETIME)

### `about`
- `id` (BIGINT, Primary Key)
- `introduction` (TEXT)
- `career_objective` (TEXT)
- `education_details` (TEXT)
- `experience_summary` (TEXT)

## Database Setup
1. Create a database in MySQL:
   ```sql
   CREATE DATABASE portfolio_db;
   ```
2. The schema will be automatically generated by Spring Boot / Hibernate using `spring.jpa.hibernate.ddl-auto=update`.
